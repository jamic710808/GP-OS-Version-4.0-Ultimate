四、DAX 公式完整解析
以下為本課程體系中所有 DAX 公式的詳細解說，涵蓋計算邏輯、Filter Context 運作機制及常見陷阱。
4.1 核心基礎指標
① 銷售收入 (Sales Revenue)
直接加總銷售訂單表中的收入欄位，在當前 Filter Context 下自動完成篩選聚合。
銷售收入 = SUM('F銷售訂單表'[銷售金額])
② 銷售成本 (Sales Cost)
加總銷售訂單表中的成本欄位。注意：若成本欄位存在空值（Zero Inventory問題），需配合 LASTNONBLANKVALUE 修正。
銷售成本 = SUM('F銷售訂單表'[銷售成本])
③ 銷售毛利 (Gross Profit)
毛利 = 收入 − 成本，建議引用已定義的度量值而非直接計算欄位，確保 Filter Context 一致性。
銷售毛利 = [銷售收入] - [銷售成本]
④ 銷售毛利率 (Gross Margin %)
使用 DIVIDE 而非除號「/」，原因是 DIVIDE 會自動處理分母為零的情況並回傳 BLANK()，避免因零成本或零收入引發的錯誤。
銷售毛利率 = DIVIDE([銷售毛利], [銷售收入])
⚠ 常見陷阱：
不要使用 [銷售毛利] / [銷售收入]，當收入為 0 時會觸發除以零錯誤；DIVIDE 的第三參數可設定替代值，如 DIVIDE([銷售毛利], [銷售收入], 0)。
4.2 加權平均採購成本 (Weighted Average Purchase Cost)
這是最容易犯錯的指標之一。錯誤做法是直接用 AVERAGE(單價)，這完全忽略了「採購數量」的權重，導致計算結果失真。
❌ 錯誤做法（簡單平均 - WRONG）
平均採購成本_錯誤 = AVERAGE('F存貨變動表'[單位成本])

// 例：採購1=800件@$135，採購2=200件@$174
// 錯誤結果：($135+$174)/2 = $154.5  ← 偏高！
✅ 正確做法（加權平均 - CORRECT）
平均採購成本 = DIVIDE(
    SUM('F存貨變動表'[采購入庫金額]),
    SUM('F存貨變動表'[采購入庫數量])
)

// 正確結果：(800×$135 + 200×$174) / (800+200) = $142.8
Filter Context 說明：SUM 在當前上下文中自動篩選，無需額外 CALCULATE，因為事實表中的篩選已由 Slicer 或 Row Context 決定。
4.3 零庫存成本修正 (LASTNONBLANKVALUE)
業務問題場景：當某月有銷售但 ERP 系統中無期末庫存時，「成本 = 銷售數量 × 單位成本」公式會因單位成本為空而導致銷售成本為 0，不符合業務邏輯（ERP 應繼承上一個有庫存月份的單位成本）。
// 解決方案：使用 LASTNONBLANKVALUE 向前填充最後一個有效成本
優化後單位成本 = 
    CALCULATE(
        LASTNONBLANKVALUE(
            日期表[日期],       -- 在日期序列上搜索
            [單位成本]           -- 目標度量值
        ),
        FILTER(
            ALL(日期表),         -- 移除日期的 Filter Context
            日期表[日期] <= MAX(日期表[日期])  -- 只看截至當前日期
        )
    )
4.4 預警毛利率連續下跌天數
商業意義：持續的價格折扣會讓消費者產生等待更低價的預期心理。監控連續下跌天數，是為了防止折扣策略失控，侵蝕品牌價值和長期利潤。
連續下降天數 = 
VAR X = MAX(日期表[日期])              -- Step1: 定義當前日期
VAR 所有日期 = VALUES(日期表[日期])
VAR 下跌日期 = FILTER(所有日期, [毛利率] < EARLIER([毛利率]))  -- 所有毛利率下跌日
VAR AA = EXCEPT(所有日期, 下跌日期)   -- Step2: 找出上漲或持平的日期
VAR BB = MAXX(
    FILTER(AA, [日期列值] < X),        -- Step3: 離當前最近的上漲日
    [日期列值]
)
RETURN INT(X - BB)                    -- Step4: 計算天數差
陷阱：矩陣總計錯誤 (Grand Total Bug)
問題根源：在沒有行上下文 (Row Context) 時，度量值在總計行計算整個期間的最大值，而不是明細行的加總，導致總計與明細對不上。
// ❌ 問題：總計行顯示整個期間最大連續下跌天數 (如5)
//    而非期末的實際值 (如2)

// ✅ 修正：使用 MAXX + VALUES 黃金組合
連續下降天數_修正 = 
    MAXX(
        VALUES(日期表[日期]),          -- 在總計行返回所有日期列表
        [連續下降天數原始]              -- 對每個日期迭代計算後聚合
    )
4.5 SWITCH(TRUE) 多條件分類警戒線
應用場景：不同產品類別有不同的行業基準毛利率，需為每個品類設定專屬的警戒閾值，而非使用統一門檻。
// 簡易版 SWITCH 警戒線
當前品類警戒線 = 
VAR SelectedCategory = SELECTEDVALUE('D產品表'[產品類別])
RETURN
    SWITCH(TRUE(),
        SelectedCategory = "服飾類", [服飾類警戒線·值],   -- 條件1→結果1
        SelectedCategory = "飾品類", [飾品類警戒線·值],   -- 條件2→結果2
        SelectedCategory = "鞋類",   [鞋類警戒線·值],     -- 條件3→結果3
        [通用警戒線·值]                                     -- Else 默認值
    )
進階版：結合 SUMMARIZE + FILTER 計算低毛利銷售
// 完整進階版：計算低於品類警戒線的平均毛利率
VAR A  = SELECTEDVALUE('D產品表'[產品類別])
VAR X1 = [服裝類毛利率警戒線·值]
VAR X2 = [飾品類毛利率警戒線·值]
VAR AA = SWITCH(TRUE(),
    A = "服裝類",
        VAR Y = SUMMARIZE('F銷售訂單表',
                    'D產品表'[產品名稱],
                    'F銷售訂單表'[銷售單價])   -- 建立虛擬彙總表
        VAR Z = FILTER(Y, [銷售毛利率] <= X1) -- 篩選低於警戒線的行
        RETURN CALCULATE([銷售毛利率], Z),    -- 在篩選上下文中計算
    A = "飾品類",
        VAR Y = SUMMARIZE('F銷售訂單表',
                    'D產品表'[產品名稱],
                    'F銷售訂單表'[銷售單價])
        VAR Z = FILTER(Y, [銷售毛利率] <= X2)
        RETURN CALCULATE([銷售毛利率], Z),
    BLANK()
)
Filter Context 說明：SUMMARIZE 建立的是虛擬表，FILTER 對虛擬表進行二次過濾，CALCULATE 將過濾後的表作為新的上下文傳入，整個流程不改變外部 Slicer 的篩選效果。
4.6 CONCATENATEX 異常日期追蹤
應用場景：當需要識別並列出某個產品低於警戒線銷售的「所有問題日期」時，傳統度量值只能返回單一數值，CONCATENATEX 可將多行文本串接成單一字串。
低毛利銷售日期清單 = 
CONCATENATEX(
    CALCULATETABLE(                          -- Step1: 建立問題日期虛擬表
        VALUES('F銷售訂單表'[訂單時間]),     -- 取出唯一日期值
        Z                                    -- Z = 低毛利銷售的過濾條件表
    ),
    [訂單時間],                              -- Step2: 要串接的欄位
    ", ",                                  -- Step3: 分隔符
    [訂單時間]                               -- Step4: 按日期升序排列
)

// 輸出範例：
// "2018/11/09, 2018/11/12, 2018/11/15, 2018/11/19, 2018/11/21..."

6.1 核心 KPI 指標卡（多行卡片圖）
6.2 三維散點圖（價格×數量×毛利率）
目標：找到能讓整體毛利最大化的「最佳價格點」。散點圖同時展示三個維度：X 軸為銷售單價、Y 軸為銷售數量、氣泡大小為銷售毛利率，色彩深淺也按毛利率高低分級（深色 = 高毛利率）。
重要洞察：追求最高「毛利率」不等於賺最多「錢」！目標是找到毛利「額」的峰值——即組合圖中絕對值柱狀圖的最高點所對應的定價。
6.3 組合圖（毛利率 vs 毛利額權衡）
用柱狀圖顯示不同定價下的毛利絕對值，用折線圖顯示毛利率百分比，雙軸呈現，幫助決策者找到毛利額峰值點（而非毛利率最高點）。
6.4 動態警戒線（Parameter + SWITCH）
在 Power BI「建模」→「新參數」中建立毛利率警戒線參數（小數，0~1，增量0.01），產生一個可互動調整的 Slicer。結合 SWITCH 函數，可為不同品類設定專屬警戒閾值，報表即時響應，標示出需要關注的低毛利產品及對應的問題日期。
